# 产品概览

`gatepay-local-mcp` 是一个本地运行的 `stdio` MCP 服务，用来让 AI 客户端访问受 `x402` 支付保护的 HTTP 接口。它对外只暴露一个请求工具，但会在服务端返回 `402 Payment Required` 时自动完成支付要求解析、EVM 签名和重试。

## 核心能力

- 通过统一的 `x402_request` 工具发起一次受支付保护的 HTTP 请求。
- 在 `402` 响应出现后，自动解析支付要求、生成支付载荷并带签名重试。
- 同时支持本地私钥签名和远程托管钱包签名两种支付执行模式。
- 将 `x402` 核心流程内置在仓库中，避免依赖外部未发布的 `@x402/*` 包。
- 以 MCP 服务形式接入 Cursor、Claude Desktop 等支持 MCP 的 AI 客户端。

## 目标使用场景

- 让 AI Agent 访问按次付费或按资源付费的 API，而不是只访问公开接口。
- 在本地开发环境中验证 `x402` 支付链路、钱包认证流程与远程服务连通性。
- 为上层 Agent 或 Skill 提供稳定的“付费请求桥接层”，减少每次调用时重复处理支付协议的成本。

## 价值主张

该项目的价值不在于提供大量 MCP 工具，而在于把复杂的 `x402` 支付细节收敛到一个最小而稳定的调用面。对接方只需提供完整 URL、方法和可选 JSON 请求体，就能复用统一的支付、签名与重试流程。

另一个关键价值是双钱包模式：优先支持本地 EVM 私钥的直接签名，同时保留基于远程 MCP 钱包的托管登录与签名能力，便于在不同安全边界和运行环境下复用同一套请求接口。
